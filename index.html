AutoDS

"A local-first AI data science assistant that converts natural language analysis requests into controlled, executable data-science workflows using an MCP-style tool server."

Problem Statement

Data scientists spend hours writing repetitive analysis code. Users want to ask questions like "show me correlations" or "train a model" but need technical skills to write Python scripts. There's no easy way for non-technical users to perform data analysis without learning programming.

Literature Review / Market Research

Tools like AutoML (Google AutoML, H2O) are cloud-based and expensive. Jupyter notebooks require coding skills. ChatGPT can explain analysis but can't actually execute it on your data. No local solution exists that combines natural language with actual data analysis execution.

Research Gap / Innovation

AutoDS runs completely locally using Ollama (no cloud needed). It separates AI decision-making from actual execution using MCP architecture. User talks to AI in plain English, AI decides what to do, MCP server safely executes data analysis. This makes data science accessible to everyone.

System Methodology

Dataset / Input

Users upload CSV datasets through simple interface. Natural language requests like "analyze this data" or "show correlations". System accepts standard tabular data formats.

Model / Architecture

Three layers: UI Layer (Streamlit/CLI), AI Agent (Ollama for understanding), MCP Server (pandas, sklearn, matplotlib for execution). Each layer has one job and works independently.

System Architecture

UI Layer

Simple interface (CLI or Streamlit) where users upload datasets and ask questions in plain English

AI Agent (Ollama)

Understands what user wants, decides which analysis tool to use, sends commands to MCP server

MCP Server

Executes actual data analysis using pandas, matplotlib, scikit-learn. Returns results safely

Challenges Faced

• Getting Ollama to understand diverse natural language queries consistently
• Handling different dataset formats and missing data gracefully
• Balancing simplicity for users with powerful features for advanced users
• Making visualizations clear and informative without overwhelming users

Key Learnings

• Importance of clear separation between AI decision-making and actual execution
• Local LLMs like Ollama are powerful enough for intelligent tool routing
• Modular tool design makes the system easy to expand and maintain
• Simple interfaces with smart backend make data science accessible to everyone
